# BotMentor — Safety & Ethics (plain language)

The Claude Corps program asks fellows to use AI **effectively, efficiently, ethically, and
safely**. Here is how BotMentor meets that, in plain terms.

## Keeping students safe (Safety)
- The tool adds **safety reminders** to its answers — for example, "have an adult present for
  battery or soldering work." Robotics involves batteries, sharp parts, and tools; the tool
  is built to slow a student down at the right moments.
- The tool **never** tells a student to do something dangerous, and it pushes physical testing
  to the student (who is supervised) rather than attempting anything itself.

## Being honest about limits (Ethics + Discernment)
- When the tool isn't confident, it **says so** instead of guessing.
- It ranks causes by likelihood and flags the **cheapest, safest check first**, so students
  don't waste money swapping parts or take avoidable risks.
- If the tool can't understand the question, it asks the student to describe the problem
  better (the "Description" step) rather than inventing an answer.

## Using resources well (Efficiency)
- In practice mode ("mock"), the tool costs **nothing** and needs no paid key — good for
  classrooms and events with no budget.
- When using real AI, each answer is a fraction of a cent; we document the expected monthly
  cost so the organization can plan.

## Doing the job well (Effectiveness)
- The tool is **coaching**, not fixing — it builds the student's own problem-solving skill,
  which is the lasting outcome a mentor at an expo would aim for.
- The whole point of the two-guide handoff (this non-technical set + the developer runbooks)
  is **effectiveness that outlives the original builder** — the core Claude Corps promise.

## Privacy
- The tool does **not** require students to log in or share personal information. What a
  student types is sent to the AI to generate an answer; we recommend the organization review
  the AI provider's data handling if student data sensitivity becomes a concern.
