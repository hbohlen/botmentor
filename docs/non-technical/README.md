# BotMentor — Non-Technical Documentation

This folder is the **plain-language track** for people who will help own or grow BotMentor
without writing code: program leads, teachers, volunteer coordinators, and stakeholders.
The technical runbooks and architecture notes live in the **parent `docs/` folder** for
developers.

Read in this order:

1. **[00-plain-language-guide.md](00-plain-language-guide.md)** — *What is BotMentor, what does
   it cost, what do I do day-to-day, and what if it breaks?* Start here.
2. **[01-orientation.md](01-orientation.md)** — a 2-minute briefing you can read at a kickoff
   or hand to a new volunteer.
3. **[02-safety-and-ethics.md](02-safety-and-ethics.md)** — how the tool meets the Claude Corps
   "effective, efficient, ethical, safe" standard, in plain terms.

**How the two tracks fit together (the handoff design):**
- *Technical readers* → `docs/02-runbooks.md`, `docs/03-maintain-and-extend.md`,
  `docs/04-architecture-decisions.md`.
- *Non-technical readers* → **this folder**.
- Both describe the **same system**; the split is audience, not content. A non-coder can own
  and grow the project (add a coaching tip, translate the UI, run a workshop) without ever
  touching the code — which is exactly the "make sure it survives you" requirement.
